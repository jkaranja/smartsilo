Key flows

Inbound OAuth — MCP client authorizes against our system
1
Claude Desktop hits mcp/.well-known/oauth-protected-resource → discovers auth server URL
↓
2
Client fetches api/.well-known/oauth-authorization-server → gets token & auth endpoints
↓
3
Client registers dynamically at api/oauth2/register (or uses URL-based client ID from api/mcp/client)
↓
4
User redirected to web/sign-in?oauth_query=… → authenticates
↓
5
Consent page at web/consent → user approves scopes → api/oauth2/consent
↓
6
Client exchanges code → api/oauth2/token → receives JWT access token
↓
7
Client sends Bearer <token> to mcp/mcp → mcpHandler verifies against api/jwks → tools served
Outbound OAuth — our agent connects to external MCP server
1
Admin adds server via POST api/mcp — tools fetched and stored in McpServer.tools
↓
2
POST api/mcp/:id/authorize → discovers external auth server via RFC 9728
↓
3
Registers as OAuth client (DCR) or uses api/mcp/client URL as client ID (SEP-991)
↓
4
PKCE code verifier stored → authorization URL returned to admin UI → user opens in browser
↓
5
External service redirects to api/mcp/callback → code exchanged for tokens
↓
6
Tokens stored in McpServer.authToken / refreshToken — agent uses on next connection
WebSocket agent loop — user sends a message
1
WS upgrade: authenticate session, load McpServer rows with cached tools JSON from DB
↓
2
agent.setServers(configs) — stores configs, zero live connections made
↓
3
User sends message → agent.run({ tools }) — all tools passed from DB, no live fetch
↓
4
Anthropic streams response — text deltas yielded to client in real time
↓
5
Model calls a tool → lazyConnect(serverUrl) — first real MCP connection, cached for session
↓
6
Tool result fed back → model continues → end_turn → messages saved to AgentThread
SSO sign-in — enterprise user logs in
1
Org admin registers IdP at web/settings/organization → POST api/settings/sso
↓
2
Better Auth sso() plugin stores provider · discovers OIDC config from issuer
↓
3
User enters work email on web/sign-in → authClient.signIn.sso({ email })
↓
4
Better Auth matches email domain → redirects to org's IdP (Google Workspace, Okta, Azure AD…)
↓
5
IdP callback → organizationProvisioning auto-assigns user to org with configured role
